Entry Point - App.tsx

El archivo app.tsx muestra la estructura básica de la aplicación construida con React, la cual utiliza diversas bibliotecas y componentes para implementar funcionalidades como autenticación de usuarios, enrutamiento, protección de rutas, internacionalización y más. Cada parte del código cumple un propósito específico para lograr la funcionalidad deseada en la aplicación. Al entender la estructura y la funcionalidad de las diferentes partes, se puede trabajar en el desarrollo y mantenimiento de la aplicación de manera más efectiva.

Componente App​

El componente App es el componente principal de la aplicación. Contiene el enrutamiento de las diferentes pantallas y proporciona el contexto de usuario a través del componente UserContext.Provider. También se encarga de cargar los archivos de idioma y configurar el proveedor de internacionalización (IntlProvider) para admitir la traducción de la interfaz de usuario.

CssBaseline​

El componente CssBaseline es parte del conjunto de componentes proporcionados por Material-UI, el cual se utiliza para establecer un conjunto básico de estilos CSS en la aplicación y normalizar el aspecto visual en diferentes navegadores.

Internacionalización (IntlProvider)​

El componente IntlProvider se utiliza para admitir la internacionalización de la aplicación. Se configura con el idioma actual y los mensajes de traducción correspondientes cargados desde archivos JSON (en.json y es.json). Esto permite que los componentes utilicen las funciones de formateo y traducción proporcionadas por la biblioteca react-intl para mostrar el contenido en el idioma seleccionado.

Contexto de usuario (UserContext)​

El contexto de usuario (UserContext) se define en el archivo UserContext.tsx y se exporta junto con el componente App. Este contexto proporciona información relacionada con el usuario, como los detalles del usuario autenticado, el estado de carga del usuario, el servicio BackendServices, el idioma actual, etc. Este contexto se utiliza en diferentes componentes de la aplicación para acceder a la información del usuario de manera sencilla.

Componente NavigationContainer​

El componente NavigationContainer es un componente envolvente que proporciona una estructura de navegación común para todas las pantallas. Es responsable de mostrar una barra de navegación o menú para que el usuario pueda navegar entre las diferentes pantallas de la aplicación.

Componentes de pantalla y enrutamiento​

El componente App define diferentes rutas utilizando el enrutador de React (BrowserRouter y Routes) para cada pantalla de la aplicación. Algunas de las pantallas mencionadas incluyen BudgetScreen y LoginScreen. Cada ruta se mapea a un componente de pantalla correspondiente que se renderiza cuando se accede a esa ruta en la aplicación.

Componente PrivateRoute​

El componente PrivateRoute es un componente personalizado que se utiliza para proteger las rutas y garantizar que solo los usuarios autenticados tengan acceso a ciertas pantallas. Este componente envuelve las rutas que requieren autenticación y redirige al usuario a la pantalla de inicio de sesión si no está autenticado.

/src/app.tsx
const App = () => {

    return (
        <>
            <CssBaseline />
            <IntlProvider locale={locale} messages={loadLocaleData(locale)}>
                <UserContext.Provider value={{ user: userInfo, userLoading, backendServices, locale, setLocale, loading, setLoading }}>
                    <BrowserRouter>
                        <NavigationContainer>
                            <Routes>
                                <Route path="/" element={<PrivateRoute element={<BudgetScreen userInfo={userInfo} />} />} />
                                <Route path="login" element={<LoginScreen />} />
                            </Routes>
                        </NavigationContainer>
                    </BrowserRouter>
                </UserContext.Provider>
            </IntlProvider>
        </>
    )
}
export default App

Hook useEffect para cerrar sesión por inactividad​

Dentro del componente App, se utiliza un hook useEffect(checkInactivity) para agregar un temporizador que cierra la sesión del usuario después de 30 minutos de inactividad. Esto se logra mediante el seguimiento del tiempo de la última actividad del usuario y verificando si ha pasado el tiempo de inactividad especificado. En caso de inactividad, se cierra la sesión del usuario utilizando el servicio auth de Firebase y se recarga la página.

/src/app.tsx
useEffect(() => {
    const checkInactivity = () => {
        const currentTime = Date.now();
        const timeSinceLastActivity = currentTime - lastActivity;
        if (timeSinceLastActivity >= 30 * 60 * 1000) {
        console.log('No activity in the last 30 minutes, logging out user');
        auth.signOut().then(() => {
            window.location.reload()
            console.log('User logged out');
        }).catch((error) => {
            console.log(error.message);
        });
        }
    };
    
    const resetTimer = () => {
        setLastActivity(Date.now());
    };
    
    // document.addEventListener('mousemove', resetTimer);
    document.addEventListener('keydown', resetTimer);
    // document.addEventListener('scroll', resetTimer);
    
    const timer = setInterval(checkInactivity, 1800000);
    
    return () => {
        clearInterval(timer);
        document.removeEventListener('mousemove', resetTimer);
        document.removeEventListener('keydown', resetTimer);
        document.removeEventListener('scroll', resetTimer);
    };
}, [auth, lastActivity]);

Hook useEffect para obtener información del usuario​

Hook useEffect que se utiliza para obtener información del usuario actualmente autenticado. Se suscribe al evento onAuthStateChanged del servicio auth de Firebase para recibir actualizaciones cuando el estado de autenticación cambia. Dentro de este hook, se obtiene el token de autenticación del usuario y se utiliza para inicializar el servicio BackendServices. Luego, se obtiene la información adicional del usuario y se actualiza el estado de userInfo. También se establece el idioma según la configuración actual.

/src/app.tsx
useEffect(() => {
    console.log('init')
    console.log('retrieving user')
    const unsubscribeCallback = auth.onAuthStateChanged((user) => {
        console.log(user)
        if (!user) {
            console.log('no user')
            setUserLoading(false)
            return
        }
        console.log('starting back connection')
        user?.getIdToken().then((token) => {
            const backServices = new BackendServices(token)
            setBackendServices(backServices)
            console.log('retrieving user info')
            backServices.getUserInfo().then((result) => {
                setUserInfo({
                    ...result.data,
                    photoURL: user?.photoURL,
                })
            })
            setUserLoading(false)
        })
        if (locale === 'es') {
            setLocale('es')
        } else {
            setLocale('en')
        }
    })
    return () => {
        unsubscribeCallback()
    }
}, [])
