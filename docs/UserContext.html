Contextos

UserContext​

El archivo define un contexto llamado UserContext (utilizando la función createContext de React) que se utiliza para compartir información relacionada con el usuario y la configuración en toda la aplicación. Un contexto es una forma de compartir y acceder a datos entre componentes, sin la necesidad de pasar props manualmente a través de cada nivel de la jerarquía de componentes. En este caso UserContext proporciona valores predeterminados que pueden ser accedidas y modificadas por los componentes que consumen este contexto.

Definir propiedades para implementar en el contexto​

<user> <userLoading> <backendServices> <locale> <setLocale> <loading> y <setLoading>

Estas son las propiedades definidas en el objeto con un valor predeterminado en el contexto. Cada propiedad representa un valor relacionado con el usuario y la configuración de la aplicación.

  • user: Representa la información del usuario actualmente autenticado.
  • userLoading: Indica si la información del usuario está cargando o no.
  • backendServices: Representa los servicios backend utilizados en la aplicación.
  • locale: Representa el idioma actual de la aplicación.
  • setLocale: Una función que se utiliza para cambiar el idioma de la aplicación.
  • loading: Indica si la aplicación está cargando o no.
  • setLoading: Una función que se utiliza para cambiar el estado de carga de la aplicación.
/src/contexts/UserContext.tsx
import { createContext } from 'react'
import BackendServices from '../services/BackendServices'

const UserContext = createContext<{
    user: UserInfo | undefined
    userLoading: boolean
    backendServices?: BackendServices
    locale: 'en' | 'es'
    setLocale: (locale: 'en' | 'es') => void
    loading: boolean
    setLoading: (loading: boolean) => void
}>({
    user: undefined,
    userLoading: true,
    backendServices: undefined,
    locale: 'en',
    setLocale: (locale) => {
        console.log(`Cambio de locale a ${locale}`)
    },
    loading: false,
    setLoading: (loading) => {
      console.log(`Cambio de locale a ${loading}`)
    },
})

export default UserContext

Implementación global del contexto​

Se importa el UserContext y se engloba toda la aplicación. Al envolver los componentes con UserContext.Provider, se asegura de que los componentes hijos puedan acceder a las propiedades del contexto UserContext para obtener los valores requeridos. El .Provider es una propiedad especial de los componentes de contexto en React que se utiliza para definir el alcance del contexto y proporcionar los valores del contexto a los componentes hijos. Es el encargado de propagar los datos del contexto a través del árbol de componentes.

/src/app.tsx
import UserContext from './contexts/UserContext'

<IntlProvider locale={locale} messages={loadLocaleData(locale)}>
    // Implementación global del contexto
    <UserContext.Provider value={{ user: userInfo, userLoading, backendServices, locale, setLocale, loading, setLoading }}>
        <BrowserRouter>
            <NavigationContainer>
                <Routes>
                    <Route path="/" element={<PrivateRoute element={<BudgetScreen userInfo={userInfo} />} />} />
                    <Route path="login" element={<LoginScreen />} />
                </Routes>
            </NavigationContainer>
        </BrowserRouter>
    </UserContext.Provider>
</IntlProvider>
